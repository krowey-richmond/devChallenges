# DevChallenges Portfolio

This repository serves as a **central hub** for my DevChallenges projects. Each challenge lives in its own folder and is accessible through a simple, clean HTML homepage that acts as a navigation layer across all projects.

The focus of this repository is **clarity, structure, and frontend fundamentals**—showing how individual challenge solutions fit into a coherent, professional workflow.

🔗 **[View Live Demo](https://krowey-richmond.github.io/devChallenges/)**

---

## 🧭 How This Repository Works

- The root of the repository contains an `index.html` file.
- This page functions as a **mini portfolio / directory**.
- Each listed item links directly to a DevChallenges project inside this folder.
- Every project has its own self-contained structure and can be viewed independently.

This mirrors how real-world frontend work is organized: one entry point, many focused implementations.

---

## 📂 Project Structure

```txt
DevChallenge/
│
├── index.html          # Portfolio / navigation page
├── styles.css          # Styles for the portfolio page
│
├── simple-faq/
│   ├── index.html
│   ├── styles.css
│   └── assets/
│
├── blog-card/
│   ├── index.html
│   ├── styles.css
│   └── assets/
│
└── README.md
```

- Each project folder contains its **own HTML page**, styles, and assets.
- Navigation back to the main portfolio page is handled via a simple `Back` link.
